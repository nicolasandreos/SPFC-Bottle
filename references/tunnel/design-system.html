05 · Behavior

Motion & Interaction

All entrance motion is driven by GSAP + ScrollTrigger, wired to SplitType-decomposed characters and words. Hover transitions are Webflow IX or plain CSS.

letters-fade-in

Each character fades in over 0.8s (power1.out, staggered 0.8s). Triggers at 80% viewport.

Design System

words-slide-from-right

Words slide in from 1em right, fading. 0.6s · power2.out · 0.2s total stagger.

Built for your brand

scrub-each-word

Each word's opacity scrubs 0.2 → 1 with scroll. Start 90%, end center.

Scroll to reveal each word gradually

Button · gradient slide

.button-grad slides from left:-100% → 0 on hover · 0.4s · power2.out. Opacity fades in.

Hover me

Nav link · color transition

.nav-link color transitions from rgba(255,255,255,.5) → --neon-green. transition: all 0.2s

Default Hovered

Secondary · fill-invert hover

.button.is-secondary:hover — border & bg become --neon-green; color flips to --dark-green.

Hover me

Grad-line SVG entrance

.grad-line-ani-image — decorative SVGs animated by Webflow IX on scroll (translate & scale).

Poly-card · clip-path

Responsive clip-path polygon chamfers corners (24px desktop / 16px mobile). Drop-shadow on wrapper.

SplitType setup

[text-split] elements are decomposed into .word > .char spans on DOMContentLoaded before GSAP runs.

[text-split] → opacity 0
.word → overflow hidden
.char → inline-block
gsap.set → opacity 1

Navbar · collapse

.navbar.w-nav uses Webflow's built-in collapse animation on breakpoint medium.

data-duration="400"
